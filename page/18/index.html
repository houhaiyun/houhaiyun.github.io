varnishadm(命令行接口)常用子命令

命令行管理接口命令

varnishadm [-t timeout] [-S secret_file] [-T address:port] [-n name] [command[...]]

  • -S secret_file：指定秘钥文件路径(/etc/varnish/secret)
  • -T -T address:port: 指定连接命令行接口的地址和端口

默认可不指选项参数，varnishadm即可连接到命令行管理接口

Varnish简介

Varnish 是一款高性能且开源的反向代理服务器和 HTTP 加速器，其采用全新的软件体系机构，和现在的硬件体系紧密配合，与传统的 squid 相比，varnish 具有性能更高、速度更快、管理更加方便等诸多优点，很多大型的网站都开始尝试使用 varnish 来替换 squid，这些都促进 varnish 迅速发展起来。

挪威的最大的在线报纸 Verdens Gang(vg.no) 使用 3 台 Varnish 代替了原来的 12 台 Squid，性能比以前更好，这是 Varnish 最成功的应用案例。

http cache 解决方案

squidVSvarnish

  • 基于http协议提专业的供缓存控制的程序有squid和varnish，squid比较重量级，是个老家伙，不过，老当益壮。
  • 在高并发的情况下仍能保持较好的稳定性，而varnish比较轻量，也是比较young的一款缓存管理程序，性能与squid不相上下。
  • 而在高并发的情况下，varnish可能就没有squid稳定，不过又有使用者反映varnish的性能和稳定性高于squid，不管是否如此，两款开源程序都是不错的http缓存解决方案，本文仅以varnish作为学习对象。

Cache缓存相关

本文中描述的缓存概念多是相对http缓存而言。

缓存作用

缓存所起的作用就是加速，减轻后端服务器压力。一般而言，我们请求的大多数web资源往往不是由后端服务器所响应的(对于颇具规模的站点来说)，而是由前端的缓存服务器所缓存的内容直接响应，所以，我们得到的资源可能不实最新的，因为缓存的数据于后端服务器不一定时同步更新的。
